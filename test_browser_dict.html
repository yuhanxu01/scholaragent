Global Dictionary Feature Test

Instructions:

1. Make sure the frontend server is running on http://localhost:5173

2. Log in to the application

3. Navigate to any page (Dashboard, Documents, etc.)

4. Double-click on any English word to see the dictionary popup

5. In the popup, double-click words in definitions, translations, or examples to continue looking up words

Test Text

Here is some sample text for testing the global dictionary feature. The implementation provides comprehensive functionality for vocabulary acquisition. This sophisticated system offers seamless integration with existing applications.

Technology encompasses various aspects of modern life. Software development requires continuous learning and adaptation to new paradigms. The architecture supports multiple interaction patterns.

Expected Behavior

  • ✅ Double-clicking any English word (2+ characters) should show a dictionary popup
  • ✅ Popup should appear near the cursor position
  • ✅ Popup should show word definition, translation, examples, and pronunciation
  • ✅ Can add words to vocabulary collection
  • ✅ Can play word pronunciation
  • ✅ Double-clicking words in the popup results should open new lookups
  • ✅ Clicking outside or pressing ESC closes the popup
  • ✅ Won't trigger on input fields, buttons, or other interactive elements

Test Cases to Try

  1. Double-click "implementation" - should show definition and examples
  2. Double-click "comprehensive" - should show related words suggestions
  3. Double-click "technology" - should show pronunciation button
  4. In the popup, double-click words in the definition to continue looking up
  5. Try adding a word to vocabulary collection
  6. Test on different pages: Dashboard, Documents, Notes, etc.

Known Limitations

  • Words must be at least 2 characters long
  • Pure numbers are ignored
  • Doesn't trigger on input fields, buttons, select options
  • Doesn't work inside the dictionary popup itself (to prevent loops)
